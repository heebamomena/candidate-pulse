# Candidate Pulse

An interactive tool that explores how candidates experience the hiring process.

## 💡 Why I built this

Having worked in HR operations and recruitment, and now being a candidate myself, I noticed a gap:

We track hiring metrics, but we don’t always understand how candidates actually experience the process.

Candidate Pulse is an attempt to explore that gap.

## 🔍 What it shows

- Where candidates drop off in the hiring funnel  
- How communication impacts candidate sentiment  
- What small improvements can improve hiring outcomes  

## ⚙️ How it was built

This project was built using an AI-assisted approach often called “vibe coding” — describing the idea and letting AI help generate the structure and code.

The focus was on exploring the problem, not writing everything manually.

## 📊 Data

All data used in this project is mock data, designed to simulate real hiring scenarios.
